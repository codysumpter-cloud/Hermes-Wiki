## 2. gateway multiplex phase 0-4

> **背景**：v0.13.0 引入 profile 隔离时，每个 profile 都需要起独立 gateway 进程（每个有自己的 PID、port、log dir）。Phase 0-4 让一个 gateway 进程内并行多个 profile 的 messaging adapters，HTTP webhook 经 `/p/<profile>/` 前缀路由；每个 profile 的 adapter 跑在独立的 secret scope 里（fail-closed 凭据隔离）；启动时检测端口绑定 / 凭据冲突；可观测性写入 served_profiles。
>
> **状态**：opt-in（默认 `gateway.multiplex_profiles=false`），实验性。需要 `hermes config set gateway.multiplex_profiles true`。
