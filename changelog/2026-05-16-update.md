# Hermes Agent 更新日志（2026-04-24 ~ 2026-05-16）

自上次同步（v2026.4.23 / v0.11.0）以来 **2890 个 commit**，跨越三个版本：

| 版本 | 标签 | 日期 |
|---|---|---|
| v0.12.0 | v2026.4.30 | 2026-04-30 |
| v0.13.0 | v2026.5.7 | 2026-05-07 |

这是 wiki 跟踪以来**最大的一次更新窗口**。核心主题：**Provider 全面插件化**（model-provider + web-search 两大体系）、**Kanban 协作看板**（全新多 Agent 运行时）、**Goals / Ralph Loop**、**LSP 语义诊断**、**新平台/工具/Provider 大批接入**。

## 架构级变更

### 1. Web 搜索后端插件化

旧的 `tools/web_providers/` 目录被删除。7 个搜索后端（firecrawl / tavily / parallel / exa / searxng / brave_free / ddgs）迁移为 `plugins/web/` 下的插件。

- 新 ABC `agent/web_search_provider.py`（221 行）：`WebSearchProvider`，能力标志 `supports_search/extract/crawl`，`extract`/`crawl` 可为 async。
- 新 registry `agent/web_search_registry.py`（262 行）：**按能力**分别解析 active provider。
- 新门面 `ctx.register_web_search_provider()`。

详见 [[web-tools-architecture]]。

### 2. Model Provider 插件化（33 个 provider）

`providers/` 包和 `plugins/model-providers/` 目录在 v0.11.0 时**都还不存在**——v0.12.0–v0.14.0 把全部 33 个 provider 改造成声明式插件。

- `providers/base.py`（184 行）：`ProviderProfile` dataclass，描述认证/端点/请求怪癖。
- transport 改为**单路径**——读 ProviderProfile 而非 20+ 布尔 flag。
- `providers/__init__.py` 三层发现：bundled → 用户目录 → 旧式单文件模块。

详见 [[smart-model-routing]]。

### 3. Kanban 协作看板（全新多 Agent 运行时）

`feat(kanban): durable multi-profile collaboration board #17805`——持久化、SQLite 支撑、跨进程的多 worker 协作看板。

- DB 内核 `hermes_cli/kanban_db.py`（约 4839 行），任务生命周期 `triage→todo→ready→running→blocked→done→archived`。
- 工具集 `tools/kanban_tools.py`（1139 行，9 个工具，普通 chat 下 schema 占用为零）。
- 诊断引擎 `hermes_cli/kanban_diagnostics.py`、幻觉门、Dashboard 插件。

成为 Hermes 第 5 种多 Agent 机制。详见新页 [[kanban-system]]。

### 4. Goals / Ralph Loop

`feat: /goal — persistent cross-turn goals (Ralph loop) #18262`——跨轮次持久目标，judge 模型每轮判定是否达成，未达成则注入 continuation prompt 继续。`hermes_cli/goals.py`（756 行）。详见新页 [[goals-and-ralph-loop]]。

## 新增工具与能力

### 视频工具

- **`video_generate`**（`tools/video_generation_tool.py`，561 行）：统一文生视频 + 图生视频工具，可插拔后端 ABC `VideoGenProvider`。后端插件 `plugins/video_gen/`：`fal`（Veo 3.1 / Kling / Pixverse）、`xai`（Grok-Imagine）。
- **`video_analyze`**（`tools/vision_tools.py`）：把视频 base64 编码为 `video_url` content part 发给多模态 LLM（50MB 硬上限）。

### LSP 语义诊断

`agent/lsp/`（11 文件，约 4263 行）在 git 工作区内 spawn 真实语言服务器（pyright / gopls / rust-analyzer / typescript-language-server 等），`write_file` / `patch` 后通过 `lsp_diagnostics` 字段返回**新增**诊断（`range_shift` 做 delta 过滤，默认仅 ERROR）。另加 Python/JSON/YAML/TOML 进程内 linter。

### Computer Use

`tools/computer_use/` 新增 **cua-driver 后端**（`cua_backend.py`，682 行，macOS-only，MCP over stdio 接 trycua 的 cua-driver）——用私有 SkyLight SPI 实现**后台 focus 不抢占**用户光标/键盘。通用 schema（`schema.py`）支持任意工具能力模型，capture 模式 `vision`/`ax`/`som`。

### 图像多模态路由

`agent/image_routing.py`（301 行）：`decide_image_input_mode()` 按模型 `supports_vision` 元数据决定原生 `image_url` 还是文本描述；`agent.image_input_mode` 可配 `auto`/`native`/`text`。

### Checkpoints v2

`tools/checkpoint_manager.py` 重写为**单 store**架构——所有项目共用 `~/.hermes/checkpoints/store/` 一个 git 对象库，per-project 状态存为 `refs/hermes/<hash>` 分支（旧版每个 workdir 一个约 40MB 影子库）。新增真实 pruning + 磁盘上限（`max_total_size_mb`），启动时幂等 auto-prune。

## 平台/CLI/TUI

- **`hermes -z <prompt>`**：one-shot 模式（`hermes_cli/oneshot.py`），支持 `--model`/`--provider` 与 `HERMES_INFERENCE_MODEL` 环境变量。
- **TUI esbuild bundle**：TUI 改用 esbuild 打包，运行时不再依赖 `node_modules`；从 wheel 内找 bundled `entry.js`。
- **Windows 原生支持**：gateway 注册为 Scheduled Task（`hermes_cli/gateway_windows.py`），psutil 做进程管理，POSIX-only landmine 修复，skill `platforms` frontmatter gate。
- **安装/更新**：`hermes postinstall`（pip 用户）、pip `--upgrade` 支持、`ensure_dependency()` + wheel 内置 `install.sh`、`hermes update` 前自动备份 HERMES_HOME、root 安装走 FHS 布局。
- **i18n**：16 种语言（af de en es fr ga hu it ja ko pt ru tr uk zh zh-hant）。
